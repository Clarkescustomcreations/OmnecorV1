# Omnecor (CORTEX) — Full System Description

## Overview
Omnecor is a context-aware, local-first AI infrastructure designed as a unified "central nervous system" for digital and physical workflows. By consolidating software development, business automation, media generation, and hardware engineering into a single, intelligent workstation, it enables seamless multi-modal AI interaction.

## Core Architectural Pillars

### 1. Unified Backend (The Core)
Omnecor operates on a single Node.js (Express) server instance that provides a unified gateway for all interactions.
- **tRPC:** The standard for client-server communication, providing fully typed endpoints across the entire stack.
- **WebSocket (Pub/Sub):** The real-time backbone used for live state synchronization (file watchers, job progress, neural node updates).
- **Service Layer (Singleton-based):** Each major subsystem (ProcessManager, Voice, Memory, Security) exists as a managed service singleton, ensuring consistency and resource management.

### 2. The AI Provider Hub
Rather than locking into a single model, Omnecor provides a universal streaming interface for major AI providers.
- **Dynamic Routing:** A centralized `AiProviderService` routes requests to Ollama, OpenAI, Anthropic, Gemini, or Forge based on user configuration.
- **Streaming Proxy:** Built-in token-chunk streaming ensures low-latency interactivity, including custom parsers for provider-specific stream formats (e.g., Gemini's JSON array stream).

### 3. Hardware & Media Bridges (Physical Integration)
Omnecor bridges the digital AI world with physical reality using specialized Python-based microservices and bridges:
- **Blender Bridge:** Enables headless rendering and GLTF/OBJ exports via automated Python scripting.
- **KiCad Bridge:** Provides automated Design Rule Checks (DRC), Electrical Rule Checks (ERC), and BOM generation from schematics.
- **ESPTool Bridge:** Manages serial discovery and firmware flashing for IoT development.
- **Voice Pipeline:** A dedicated proxy layer orchestrates audio workflows:
    - **Whisper:** Real-time speech-to-text.
    - **TTS (XTTS-v2):** High-fidelity voice cloning.
    - **RVC:** Retrieval-based Voice Conversion for real-time conversion.

### 4. Memory & Context Architecture
Omnecor manages context through a tiered memory architecture designed to handle large-scale projects without exceeding AI token limits:
- **Layer 1 (Working Memory):** Active chat sessions (persistent in MySQL) maintained in the current context.
- **Layer 2 (Long-Term Memory):** Semantic project knowledge base powered by ChromaDB (per-project vector collections).
- **Layer 3 (Episodic Memory):** A compressed representation of past interactions, generated by AI summarization, stored in VectorDB for recall and pruning of inactive conversation history.

### 5. Neural Graph Engine
The "Brain Map" provides a spatial, interactive visualization of the project state. It is dynamically generated and synchronized in real-time via WebSocket file-system events, allowing agents and users to see the structure of their code, dependencies, and process outputs.

### 6. Safety & Sovereignty
- **Local-First:** Omnecor is designed to function offline. 
- **Agentic Wallet:** Tracks compute usage (RunPod, API spend).
- **Loop Detector:** A dedicated `HashTrackerService` monitors agent actions, applying a threshold-based halting mechanism to prevent infinite recursive loops.
- **File Security:** Integrated security scanning (YARA) and AES-256-GCM encryption for all stored project assets.
